# Patient rounds — 臨床藥師查房病人管理系統

單一檔案網頁應用。部署到 GitHub Pages 後，任何電腦開同一個網址即可使用；
設定雲端後端後，多台電腦會自動同步新增與修改的資料。

---

## 一、上傳到 GitHub Pages（取得固定網址）

1. 註冊 / 登入 GitHub → 右上角 **＋ → New repository**。
   - Repository name 隨意，例如 `patient-rounds`
   - 設為 **Private**（建議，避免被搜尋到）
   - 建立。
2. 進入該 repo → **Add file → Upload files**，把這個資料夾裡的檔案拖進去：
   - `index.html`
   - `.nojekyll`
   - （`README.md` 可傳可不傳）
   - **Commit changes**。
3. repo 上方 **Settings → Pages**：
   - Source 選 **Deploy from a branch**
   - Branch 選 **main**、資料夾選 **/(root)** → **Save**。
4. 等 1–2 分鐘，Pages 會顯示網址，形如：
   `https://<你的帳號>.github.io/patient-rounds/`
   這個網址在任何電腦、手機瀏覽器都能開。
5. 手機可用瀏覽器「加入主畫面」，像 App 一樣開啟。

> 之後要更新程式，只要在 repo 重新上傳新的 `index.html` 覆蓋即可。
> 你輸入的病人資料不在程式檔裡，所以更新程式不會影響資料。

## 二、開啟跨電腦同步（重點）

GitHub Pages 只負責「放網頁」。要讓 A 電腦的修改在 B 電腦看到，
需要一個**你自己擁有的雲端資料庫**。打開網頁後點右上 **☁ 同步**，
依視窗內步驟設定，每台電腦填**相同設定 + 相同 Workspace**即可互通。

兩種選擇（視窗內都有逐步教學）：

- **Supabase（建議）**：免費、真正的資料庫，可選日本 / 新加坡機房，
  可開啟存取權限控管（RLS）。適合較正式的使用。
- **JSONBin（最快）**：約 60 秒設定完成，但屬第三方公有服務，
  **僅建議放去識別化或練習資料**。

同步邏輯：編輯後自動上傳；開啟時自動下載最新版（以最後修改時間判斷，
較新者為準）。本機同時保留離線快取，沒網路或雲端設錯也不會掉資料。
視窗下方另有「⬇ 拉取雲端 / ⬆ 推送本機」可手動處理衝突。

## 三、病人隱私（請務必閱讀）

本系統含姓名、病歷號等可識別資料。將真實病人資料放上一般公有雲，
在台灣個資法與院內資安規範下有風險。建議擇一：

1. **去識別化**：姓名、病歷號改用代號（例如 `床12-A`、`Pt-007`）。
   查房真正會用到的是 lab、腎肝功能、用藥，去識別後完全不影響使用。
2. 與**院內資訊室**確認，架設於院內伺服器或醫院核可之雲端，並開啟權限控管。

雲端金鑰只存在各台電腦的瀏覽器本機，不會寫入「匯出」的 JSON 檔。

## 四、功能摘要

- 大項：Characteristic / Renal / Liver / 病人現況 / Medication / Lab data / Daily note
- 自動計算（附計算過程彈窗）：IBW (Devine)、BMI、eGFR-CG、MDRD、CKD-EPI 2021
- 每日可加日期紀錄：體重、Scr、Intake/Output（含 I−O 差值）、肝功能、各項 lab、daily note
- 病人頂端自動警示：腎調 / 肝調 / 膽汁鬱積 / 電解質 / 感染 / GCS / 透析 / 過敏 / 待處理藥物
- 藥物分組：Antibiotic、入院後給予、OPD 可續用、OPD 不適用；可標記查房處置狀態
- 匯出 / 匯入 JSON 備份

> 本工具為輔助記錄與計算，所有臨床判斷與劑量決定仍應以醫療團隊與原始檢驗報告為準。
> 各檢驗項目參考值為常見成人值，請依貴院檢驗室實際範圍核對。
